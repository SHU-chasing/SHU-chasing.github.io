vim yes!

1. 起因

起初从某同学的壁纸上看见的复杂快捷键，深入学习以后，,便开始对这个编辑器产生了浓厚的兴趣。

程序员的手头工作基于键盘，尽可能减少鼠标的使用才是最佳的选择。

在PS、PR中，诸如图层、通道的操作，快捷键近200余个，但是其中大多数都是对于多指的操作，很多情况下还是需要鼠标才能实现，更多的是使用了左手，节约了点击鼠标右键的时间，但是对于vim，在熟悉了其基本的操作之后，便可以通过双手，实现更便捷化的操作。

Vim 之所以能得到广大厂商与用户的认可，原因在于Vim 编辑器中设置了三种模式—命令模式、末行模式和编辑模式，每种模式分别又支持多种不同的命令快捷键，这大大提高了工作效率，而且用户在习惯之后也会觉得相当顺手。

其原理示意图如下：

命令模式：控制光标移动，可对文本进行复制、粘贴、删除和查找等工作。

输入模式：正常的文本录入。

末行模式：保存或退出文档，以及设置编辑环境

2. 使用

命令模式：

操作 作用
dd 删除（剪切）光标所在整行
5dd 删除（剪切）从光标处开始的5 行
yy 复制光标所在整行
5yy 复制从光标处开始的5 行
n 显示搜索命令定位到的下一个字符串
N 显示搜索命令定位到的上一个字符串
u 撤销上一步的操作
p 将之前删除（dd）或复制（yy）过的数据粘贴到光标后面

末行模式：

命令 作用
:w 保存
:q 退出
:q! 强制退出（放弃对文档的修改内容）
:wq! 强制保存退出
:set nu 显示行号
:set nonu 不显示行号
:命令 执行该命令
:整数 跳转到该行

3. 可视模式

相较于前文所述的三大模式以外，这个模式可以说是图形化的操作扩展。

在正常模式按下v, V, +v，可以进入可视模式。同鼠标操作一般，选择文本的时候有一种鼠标选择的即视感，有时候会很方便。

以上是关于 VIM 四种模式的解读，我们在使用 VIM 操作文本的时候，编辑区底部一般都会显示当前处于什么模式下（插入模式会有 INSERT 提示，可视模式会有 VISUAL 或者 VISUAL LINE 的提示）。

VIM 有一个很重要的按键需要一开始就做出说明，那就是键盘中的 ,这个按键用来切换模式，该按键可以快速切换到正常模式.

这个按键有点特殊，它脱离了主键盘区，每次操作这个按键的时候都会有些蛮烦。估计很多使用 VIM 的人都会有这个痛点，因此有了一个解决方案，ctrl + [ 这两个按键取代 。

4. VIM 的输入模式

编辑器的主要功能就是输入文本，编辑文本。
很多编辑器在打开文件的时候就可以通过键盘录入文字，但是 VIM 有稍许的不同，刚刚接触 VIM 的时候是很难对 VIM 进行编辑操作的，只有接触久了，了解的更加深入，也就不难理解如此设计的美妙之处。

vim全面的命令很难在一文中尽叙，需要记忆的快捷键还有很多，需要多加练习。

其学习重在实践，需要经常复盘，时间久了，也就有了使用的技能，提高工作效率。写在最后，盗用某厂的一句经典口号：

“vim yes!”
